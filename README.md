### File Naming Convention

**For DSA Problems:**
- Format: `problem-number-problem-name.ext`
- Example: `0001-two-sum.cpp`, `0001-two-sum.py`, `0001-two-sum.js`
- For multiple solutions: `0001-two-sum-brute-force.cpp`, `0001-two-sum-optimized.cpp`

**For SQL Problems:**
- Format: `problem-number-problem-name.sql`
- Example: `0175-combine-two-tables.sql`
- For multiple approaches: `0175-combine-two-tables-join.sql`, `0175-combine-two-tables-subquery.sql`

**Solution Files Include:**
- Problem statement as comments
- Multiple solution approaches
- Complexity analysis
- Test cases
- Explanation of optimal solution

## ğŸ’¡ Tips for Beginners

### DSA Tips:
1. **Start with basics**: Master fundamental data structures first
2. **Practice consistently**: Solve at least one problem daily
3. **Understand, don't memorize**: Focus on understanding the approach
4. **Compare solutions**: Always look for multiple ways to solve a problem
5. **Analyze complexity**: Understanding Big O is crucial
6. **Code in multiple languages**: It deepens your understanding

### SQL Tips:
1. **Master the fundamentals**: Start with SELECT, WHERE, JOIN operations
2. **Understand query execution**: Learn how SQL engines process queries
3. **Practice window functions**: They're powerful for complex problems
4. **Optimize queries**: Learn about indexes and query plans
5. **Use CTEs**: They make complex queries more readable
6. **Test with different data**: Consider edge cases and NULL values
